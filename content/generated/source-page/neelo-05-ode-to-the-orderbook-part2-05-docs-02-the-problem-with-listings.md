# Section 2: The Problem with Listings

Selective listing systems always face the same problem. They must make decisions before they possess the full information they would actually want.

A permissionless spot DEX does not have this issue because it lists everything. An order book venue does. It must decide which markets deserve infrastructure, which ones merit operational attention, and which ones justify the cost of liquidity provision.

That necessity turns listing into a prediction problem.

The exchange is no longer simply providing a market. It is forecasting whether a market will be worth providing. Once that happens, projects begin optimizing for the signals used in the forecast. They market for appearance, fake metrics, and shape their token design around what looks listable.

This is how listing systems become distorted even when nobody is acting maliciously.
