## 6.3 For Order Book Protocols (Hyperliquid, etc.)

### 6.3.1 Complementary, Not Competitive

Order book protocols cannot bootstrap markets. A protocol that bootstraps and matures markets is **complementary**—it creates a pipeline of tokens that may eventually graduate to order books.

### 6.3.2 Data Value

Bootstrap protocols generate data: which markets mature, trading patterns, Z-Scores, etc. Order book protocols can use this data for listing decisions—moving from "vibes-based" to rule-based.

### 6.3.3 Integration Strategy

Integrating with a bootstrap protocol (e.g., accepting graduated markets, sharing listing criteria) creates a symbiotic relationship: bootstrap protocol feeds order book; order book provides terminal liquidity.
