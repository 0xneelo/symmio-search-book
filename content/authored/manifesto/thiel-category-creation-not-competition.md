# Category Creation, Not Feature Competition

Neelo's competition section warns against pretending that an intersection of buzzwords is a new market. That warning is useful for Vibe. The claim should not be "hybrid async/sync perps with a solver" as a feature intersection. The cleaner category is: perpetual markets where none can currently exist.

That distinction changes the competitive map. Mature order books are strongest where two-sided flow already exists. Pool and vault systems can provide some bootstrap liquidity but can get trapped by capital efficiency, toxic flow, or maturity limits. Vibe's category claim is upstream: create the derivative market, measure it, and help it mature.

The compendium should therefore avoid framing Vibe as a direct replacement for every existing perp venue. The stronger position is complementary category creation. Vibe competes with the absence of market structure first; mature venues become downstream partners, validators, or eventual execution layers when a market has earned that path.
