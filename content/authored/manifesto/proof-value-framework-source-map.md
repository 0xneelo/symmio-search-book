# Proof Of Value Framework Source Map

Neelo's `13-framework-value-permissionless-perps` source is the evaluation layer for Proof of Value. It asks how Vibe should be judged if permissionless perp markets are meant to create value for LPs, traders, projects, and the wider ecosystem. This page keeps that framework navigable without turning every extracted heading, table, reference list, or source-time metric into standalone public prose.

## Reading Order

Start with `authored-proof-value-question-test`, `authored-why-proof-value-matters`, and `authored-proof-value-four-constituencies`. Those pages define the evaluation question and the four constituencies that the framework keeps returning to.

For LP value, route to `authored-token-holder-inventory-alignment`, `authored-proof-value-liquidity-role-separation`, `authored-lp-profit-decomposition-map`, and `authored-risk-alignment-matrix`. The source's core LP claim is not that yield is automatic. It is that token holders, projects, solvers, and stablecoin settlement capital play different roles, so the docs must name which risk each side bears.

For trader and project value, route to `authored-trader-project-value-loop`, `authored-trader-payout-certainty`, `authored-project-token-utility-without-stablecoin`, `authored-project-participation-beyond-revenue`, and `authored-project-alternative-cost-stack`. These pages translate the framework into product-facing questions: can traders access long-tail markets, can projects make inventory useful, and can the ecosystem learn from real flow instead of only narrative demand?

For economic clarity, route to `authored-economic-clarity-for-permissionless-perps`, `authored-solver-funded-usdc-capital-loop`, and `authored-solver-sustainability-condition`. The source is strongest when it forces the question "who loses money if the trade moves sharply against expectations?" A production page should answer that with current risk-waterfall language, not vague claims about liquidity.

For comparative advantage, route to `authored-hybrid-perps-comparative-advantage`, `authored-usdc-vault-negative-feedback-loop`, and the token-margined risk pages. The framework compares hybrid Vibe-style markets against external USDC vaults and token-margined inverse designs. That comparison is useful as a capital-structure map, but estimates and structural-superiority language still need review before launch.

For validation, sustainability, and Thiel-style durability, route to `authored-proof-value-validation-sustainability`, `authored-pre-revenue-commitment-signal`, `authored-proof-value-contribution-summary`, `authored-proof-value-thesis-restated`, `authored-proof-value-ongoing-demonstration`, and the Thiel value pages. The source's mature standard is operating evidence: real flow, retention, solver performance, stress behavior, and whether markets graduate toward healthier liquidity.

## Publication Boundary

Do not publish every number or evaluative phrase in the framework as a current Vibe guarantee. The 70 percent token-depositor/project revenue-share language, partner or committed-token metrics, capital-efficiency comparisons, stress-resilience claims, risk-waterfall details, insurance or ADL behavior, and structural-superiority conclusions are source-model material until current product, risk, legal, accounting, security, and operator review approve them.

The publication-safe claim is narrower: the framework defines how to evaluate Proof of Value. It says a permissionless perp system must align LP inventory, trader access, project participation, solver risk management, economic clarity, comparative design, and sustainability evidence before it can claim durable value.
