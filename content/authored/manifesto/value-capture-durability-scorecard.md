# Value Capture Needs Durability

The Thiel-style scorecard in Neelo's Proof of Value paper is useful because it separates excitement from business durability.

Creating value is not enough. Vibe can create value by opening perp markets for gap tokens, giving projects a better use for inventory, giving traders long and short exposure, and producing listing evidence for downstream venues. Those are value-creation claims.

Capture is a different question. The protocol, solver, and operating stack must keep enough economics to remain active. The source names several possible capture surfaces: trading fees, spread revenue, protocol share, data products, and graduation or integration paths. The compendium should not collapse those into one vague "revenue" story. Each capture path has a different dependency.

## The Durability Test

Durability is the hard part. A market can grow quickly and still be fragile if it depends on subsidies, one-time attention, unpriced solver risk, or partner optics. A durable market should show repeated use, improving execution, declining residual exposure, sustained project participation, and defensible data advantages.

That is why the docs need a scorecard instead of a slogan:

- What value is created for each constituency?
- What value is captured by the protocol and solver?
- Does capture come from usage rather than extraction?
- Does the market become less fragile as it matures?
- Does the data compound into better listings, better pricing, and better trust?

If those questions are not answered, "Proof of Value" is only a narrative. If they are answered with source-backed metrics, it becomes a falsifiable operating thesis.

## Publication Boundary

Protocol share, live fee routing, solver spread economics, data-products strategy, and launch-partner metrics remain operator-review items. This page keeps the strategy logic while avoiding unaudited capture claims.
