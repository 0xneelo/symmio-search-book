# Proof Of Value Needs Four Constituencies

Neelo's Proof of Value framework makes the thesis harder to hand-wave. A permissionless perp market is not valuable just because it can create a new trading symbol. It has to create value for four constituencies at once: LPs or token depositors, traders, projects, and the broader ecosystem.

Each group wants a different thing. LPs want risk-adjusted yield on capital they are willing to deploy. Traders want market access, execution, leverage, hedging, and confidence that winning positions can settle. Projects want utility, visibility, treasury productivity, and a market path that does not begin with expensive centralized gatekeeping. The ecosystem wants better capital efficiency, price discovery, and lifecycle continuity.

The source's important claim is alignment. If one participant class benefits only because another is being quietly mispriced, the market is brittle. If more traders create more fees, more LP revenue, deeper markets, stronger project incentives, and better downstream listing evidence, the design becomes positive-sum.

## Risk Alignment

Value alignment also requires risk alignment. Token inventory risk should sit with parties that already hold or support the token. Execution and hedge risk should sit with the solver that is built to manage it. Trader directional PnL should sit with traders when flow is netted. Tail outcomes need a documented waterfall rather than hidden socialization.

That is why this framework belongs early in Volume 02. Proof of Value is not only "demand exists." It is "demand, risk, and reward are arranged so the market can keep existing."
