# The Thiel X/Y Value Frame

The Thiel frame separates two questions the docs must keep apart: how much value the system creates, and how much of that value the system can durably capture.

That distinction matters for Vibe because permissionless perp infrastructure can be useful without automatically becoming durable. A market can help traders hedge, help projects turn token inventory into utility, help LPs earn yield, and help venues see listing evidence. Those are value-creation claims. But the protocol still needs sustainable capture surfaces: fees, spreads, protocol share, data, integrations, or other economics that keep solvers, operators, and infrastructure active.

## Why This Frame Belongs In The Compendium

The source uses this framework to prevent a common documentation mistake. It is not enough to say "Vibe opens markets." The stronger claim is that Vibe may create value for multiple constituencies and capture value through the architecture that coordinates those constituencies.

That makes the page an evaluation frame, not a victory lap:

- What value is created for traders, projects, LPs, and venues?
- Which parts of that value are captured by the protocol or solver stack?
- Does capture come from real usage, or from extracting from one participant group?
- Does the market become more defensible as it grows?

The answer engine should route broad "is this valuable?" questions here before sending readers into detailed LP economics, solver sustainability, listing monopoly, or revenue pages.

## Publication Boundary

Do not treat this frame as audited proof of capture. Partner counts, protocol share, fee routing, solver spread economics, data products, and graduation economics all need current operator/product/accounting/legal review before final publication.
