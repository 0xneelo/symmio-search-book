# The Perp Bootstrap Paper Abstract

Neelo's Perp Classes / Z-Score paper starts with the market-bootstrap problem: architectures that work for established, liquid perpetual futures markets can fail when a market begins from zero.

The abstract frames the missing layer as systematic market creation. Crypto perps already matter as core trading infrastructure, but listing and market-formation processes remain manual, opaque, slow, and unable to keep up with long-tail token issuance. The paper's target is not a better homepage for existing markets. It is a method for turning first demand into a real derivative market.

The abstract then compresses the paper into three contributions. First, it classifies perpetual protocols by matching synchronicity and collateralization architecture. Second, it names the bootstrap trilemma: permissionless listing, capital efficiency, and reliable counterparty guarantees are difficult to achieve together in one static architecture. Third, it presents Vibe's hybrid lifecycle architecture as a path from collateralized asynchronous bootstrap toward more efficient synchronous order-book maturity.

## Publication Boundary

Treat the abstract as the paper's thesis map, not proof that every live Vibe market has already completed every transition. Source-time empirical examples, autonomous graduation behavior, live Z-score thresholds, and production implementation state still need fresh product/operator/risk review before public final language.
