# The Landscape Of Listing Controllers

After defining the token lifecycle, Neelo's paper turns to the protocols that control listings at different stages.

The landscape section asks three questions about each controller. How did the protocol achieve its position? Which mechanism sustains its power? How does it interact with the broader ecosystem?

That structure matters because listing power is not identical everywhere. A launchpad's power is not the same as a DEX's power. A late-stage CEX's power is not the same as a permissionless perp protocol's power. Each stage has its own gate, liquidity condition, user behavior, and moat.

## How To Read The Landscape

Read each controller as a lifecycle case study:

- **Binance** shows late-stage listing power and mature liquidity control.
- **PumpFun** shows early-stage creation and graduation control.
- **Uniswap** shows permissionless spot access and liquidity network effects.
- **Hyperliquid** shows high-performance perp infrastructure and selective market depth.

The source's goal is not just competitor description. It is to locate the missing middle by showing that existing controllers are strongest at the poles.
