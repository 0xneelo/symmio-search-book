# SYMM LP Setup Reading Boundary

Neelo's case-context section has a narrow job: it tells the reader how one SYMM LP deployment was set up before interpreting the results. It is not the performance claim itself. The source says the objective is to document the operating setup and define the context needed to read the performance metrics in the later unit-economics section.

That boundary matters because LP evidence can become misleading when setup, outcome, and generalization are collapsed into one headline. The setup page should answer who supplied the inventory, what was deposited, what market state the case observed, and which economic channels were in scope. The performance pages should then answer what happened during the measured window.

## How To Read This Part Of The Case

Read the setup section as a control surface:

- it defines the participant and deposited inventory;
- it names the market and period being studied;
- it frames the deployment as token inventory used for market formation;
- it separates context from result interpretation.

The case does not become stronger by hiding that boundary. It becomes stronger because readers can inspect the assumptions before reading the yield, benchmark, and risk pages.

## Publication Boundary

Do not use this setup section as a standalone return claim. It supports case interpretation, not generalized vault APY, audited accounting, or current product terms. Any public version should route from setup to methodology, data guardrails, risk, and replication requirements.
