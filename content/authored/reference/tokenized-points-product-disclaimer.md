# Tokenized Points Product Disclaimer

Neelo's tokenized-points-perps section starts with a hard disclaimer. The scenario is future-facing, hypothetical, not current product policy, and not an endorsed feature announcement.

That disclaimer has to stay attached to every public answer about point-linked perps. The scenario is useful because it explains how composable reward objects could become financial primitives, but usefulness is not endorsement.

## Required Labels

Any answer about tokenized point markets should preserve four labels:

- hypothetical;
- not the default intended user flow;
- third-party or community-built unless explicitly endorsed;
- not evidence of final team policy for points, packs, transferability, or derivatives support.

Those labels prevent a thought experiment from becoming a user-facing promise.

## Reader Implication

If a user asks "can I trade Vibe points as perps?", the current answer should not say yes. The safe answer is: the docs contain a hypothetical composability scenario, but it is not current product policy or an endorsed live feature.
