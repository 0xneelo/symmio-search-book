# Vibe Hotkeys

Vibe's hotkey guide treats keyboard control as part of execution speed. Hotkeys let a trader move through the ticket, select order modes, open common panels, and submit a trade without depending entirely on the mouse.

## Current Shortcut Surface

The public hotkey page lists shortcuts for Market and Limit order selection, Amount and Price input focus, token picker opening, Buy/Long and Sell/Short direction, trade placement, leverage modal opening, orderbook visibility, deposit flow, and TP/SL controls.

That means hotkeys belong in the trading workflow docs, not only in a convenience appendix. They sit directly beside order type, direction, amount, leverage, and risk-management controls.

## Guardrail

Keyboard speed does not change the trade semantics. A hotkey can move the user through the interface faster, but the same order type, margin, fee, funding, and TP/SL rules still apply. The docs should present hotkeys as UI acceleration, not as a separate execution engine.

## Reader Implication

When a trader asks how to move faster in the Vibe UI, route them here. When they ask what an order does after a shortcut selects it, route them to the simple trade flow, order-type, TP/SL, and account-health pages.
