# Referral Related Chapter Routing

The product-metrics source closes by routing readers to access phasing, points and rewards, and the conclusion.

That routing is not just an appendix. It tells the answer engine which adjacent concepts must be inspected before a metric is treated as complete.

## The Three Required Neighbors

Access phasing explains which operating state the metric belongs to. A pre-beta funnel, beta attribution metric, and open-phase conversion metric should not be compared as if they come from the same operating regime.

Points and rewards explain how referral behavior becomes economic state. Once points can affect TGE weighting, claims, packs, artifacts, or future settlement, metrics need stronger accounting boundaries.

The conclusion explains the strategic objective: referral growth should connect user acquisition, market creation, fee distribution, and trust under adversarial conditions.

## Routing Rule

When a reader asks about a referral metric, the answer should route to the metric definition first, then to the relevant chapter that owns the underlying policy:

- phase questions route to access phasing;
- points and claims questions route to points and rewards;
- trust and adversarial-condition questions route to the conclusion and security controls.

This prevents the docs from answering policy questions from a dashboard metric alone.

## Publication Boundary

This page defines routing across the referral source. It does not settle final access policy, points settlement, reward economics, referral depth, or anti-gaming enforcement.
