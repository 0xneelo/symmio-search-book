# Referral Open Participation With Optional Benefits

Neelo's long-term access phase removes hard code requirements for core participation. Codes become optional accelerators rather than mandatory gates.

That target matters because a referral program should not permanently block the product's base network effects. Gating is useful while infrastructure hardens; open participation is the eventual usability goal.

## Optional Does Not Mean Economically Irrelevant

When codes stop being mandatory, they can still route benefits, attribution, partner economics, or campaign acceleration. The difference is that a user can participate in the core product without first finding a code.

The docs should therefore avoid two misleading claims: that all code gates are permanent, or that optional codes have no economic role. The source model is phase-based.

## Publication Boundary

This page explains the source long-term target. Public referral depth is 15 levels and historical backfill is additive, but this page should not publish the exact date when code gates end, which actions become open first, which benefits remain code-linked, or whether future benefit or eligibility changes receive backfill until operator and implementation review confirm the public policy.
