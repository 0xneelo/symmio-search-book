# Referral Uniform Referee Benefit

Neelo's referral-economics source favors a uniform baseline benefit for the referred user: any valid code should grant the same baseline referee benefit.

That design changes what a user is choosing. The user is not meant to hunt for the "best discount code." Code quality should depend on issuer eligibility and attribution integrity, not on hidden variation in the referee's terms.

## Why Uniformity Reduces Code Shopping

Unequal referee benefits create a secondary market for the most generous code. Users then optimize for the discount rather than the relationship, partner, market, or source of distribution. That weakens social referrals and creates avoidable support disputes.

Uniform baseline benefits reduce that arbitrage. Commercial differences can still exist on the referrer or partner side, but the new user's baseline terms should be legible enough that onboarding does not become a code-shopping game.

## Publication Boundary

This page explains the source design principle. It should not publish final referee rakeback percentages, campaign exceptions, eligibility cutoffs, partner-specific perks, stacking behavior, or future benefit variants until operator and legal review confirm the public policy.
