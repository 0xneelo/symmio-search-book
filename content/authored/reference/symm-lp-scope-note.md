# SYMM LP Scope Note

The SYMM LP overview scope note says the case study is based on transcript statements plus dashboard data cuts. It should be treated as a first-pass operating case, not an audited performance report. The source also says core deposit and PnL values are concrete inputs in this version.

That scope note is the controlling caveat for the whole case. It lets the compendium use the case as evidence that token inventory can act as market bootstrap collateral and can produce LP-favorable economics in a specific regime, but it prevents the case from becoming a generalized vault-return claim.

## What The Scope Note Allows

The case can support mechanism-level questions:

- what inventory was deposited;
- what the market regime looked like;
- which dashboard fields were used;
- how the case window and yield methodology were defined;
- which operational lessons should shape future pilots.

It can also support a proof-of-possibility claim, if that phrase is kept narrow. The case shows that one deployment, one token, one participant, and one market regime produced a favorable data cut under the source's sign conventions.

## What The Scope Note Does Not Allow

The scope note does not allow the docs to publish audited LP APY, guaranteed yield, current vault capacity, generalized treasury allocation guidance, or final accounting treatment. It also does not replace a full NAV time series, gross-to-net attribution bridge, drawdown report, or updated benchmark across regimes.

## Publication Boundary

Any public use of the SYMM LP case should carry this boundary: first-pass operating case, transcript plus dashboard data cuts, concrete source inputs, not audited performance. Final public reporting needs source ownership, data retention, accounting review, and risk/legal/operator approval.
