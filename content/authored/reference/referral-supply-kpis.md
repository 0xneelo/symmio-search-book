# Referral Supply KPIs

Neelo's referral metrics framework treats supply as the first proof layer because Vibe needs more high-quality tradable markets, not only more users with codes.

The supply KPI set includes:

- new listings per week or day;
- active markets;
- repeat listers or weekly active listers;
- time-to-first-listing for newly onboarded creators.

Together, those metrics distinguish market creation from one-time listing bursts. A campaign can increase code issuance and first signups while failing to create durable market supply. Supply KPIs show whether referrals are actually expanding the set of assets and communities that can be traded.

## Documentation Boundary

The docs should define each supply metric with a phase label and formula version. They should not publish final listing-quality thresholds, market eligibility rules, creator scoring, listing incentives, or target bands until product and risk owners approve them.
